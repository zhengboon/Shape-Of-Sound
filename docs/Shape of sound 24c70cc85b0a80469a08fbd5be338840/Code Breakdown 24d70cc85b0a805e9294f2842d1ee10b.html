String classifyColor(int r, int g, int b) {
      int bestIndex = -1;
      int bestDist = 999999;

      for (int i = 0; i < 200; i++) {
        int dr = r - trainingRGB[i][0];
        int dg = g - trainingRGB[i][1];
        int db = b - trainingRGB[i][2];
        int dist = dr * dr + dg * dg + db * db;

        if (dist < bestDist) {
          bestDist = dist;
          bestIndex = i;
        }
      }
      return String(trainingLabels[bestIndex]);
    }

To start, we need to get the sample data of the different marbles of the 4 colors. this is achieved by using Excel Data Streamer. After recording the marble data values, they are converted back into excel. This data is here

KNN(k-Nearest-Neighbors) is a machine learning workflow. this image has been taken from https://www.ibm.com/think/topics/knn .

What this means is that our function will take the measured values of the marbles, compare it to the data stored in memory, and find the label pointing towards that data point. since we are using KNN of 7, it will compare the measured value with the closest 7 points. these 7 data points are like a vote, where the majority color will be voted as the color of the marble.

Currently, the marble classifier is about 90% accurate. There is some overlap between light green and yellow. While it is theoretically possible to make it 100%, the Arduino Nano does not have enough memory.

PROGMEM has been tested on, where the compiler stores the marble in the flash memory instead of its random access memory. While there is initial success towards 150 total variables, the Arduino begins to slow noticeably as it reaches 200. At around 250, it starts to freeze and crash.

Therefore, the Arduino Nano does not use flash memory.

For future upgrades, a dataset of 40k per color will make the accuracy 100%, assuming there is no overfitting of the function. A microcontroller that would be able to handle our specifications would be the Teensy 4.1, but it would be too expensive for our project at $50 per piece.
